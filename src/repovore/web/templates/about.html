Repovore

GitHub repository quality assessment, at a glance.

Repovore fetches metadata from any public GitHub repository, computes a composite health score from dozens of signals, and produces a structured "card" — a snapshot of a repo's quality, activity, community, and maintenance posture.

How it works

Repovore runs a four-stage pipeline on each repository. Each stage is independently resumable — if enrichment fails on one repo, it picks up where it left off.

{# Pipeline diagram — SVG #}
{# Collapsible deep-dive per stage #}
Fetch — Core repository metadata

A single GitHub API call retrieves the foundational data: name, description, stars, forks, license, topics, creation date, and last push time. This is stored as raw JSON and used as the seed for enrichment.

1 API call
13 fields captured
Stored in data/raw/
Enrich — Deep data collection

14 API calls run in parallel per repo, each fetching a different dimension of project health:

Languages Contributors (top 100) Commits (90d) Pull requests Releases Root contents Issue stats Community profile Dependabot alerts Participation stats Code frequency CI/CD runs README content Owner profile

Rate-limited to 8 requests/second with exponential backoff (2s-60s) and 3 retries. Max 5 concurrent requests.

Score — Health score computation

Four independent sub-scorers (each 0-100) are weighted and combined into a single health score:

Summarize — AI-powered analysis

An LLM (Claude) reads the scored card data plus the README and produces a structured editorial assessment:

Verdict
adopt evaluate hold avoid
Summary

3-5 sentence narrative with an opinionated take

Strengths & Concerns

2-3 concise phrases each, highlighting key positives and risks

Tags

5-10 domain/technology/use-case tags for categorization

Model: Claude Haiku 4.5 · 700 token limit · README truncated to 15,000 chars. Optional — can be disabled in config.

Quick start

1
Install 
uv sync --all-extras
Installs everything: core pipeline, web UI, and LLM summarization.
2
Set up tokens 
export GITHUB_TOKEN=ghp_...
export ANTHROPIC_API_KEY=sk-ant-...  # optional, for AI summaries
3
Fetch trending repos 
uv run repovore trending                          # this week's trending (default)
uv run repovore trending --days 1                 # last 24 hours
uv run repovore trending --days 30                # last month
uv run repovore trending --language Python        # filter by language
uv run repovore trending --topic machine-learning # filter by topic
uv run repovore trending --limit 50              # fetch up to 50 repos
Uses OSS Insight for engagement-velocity ranking, with GitHub Search API as a fallback for topic queries. Re-run periodically to keep data fresh.
4
Or process specific repos 
uv run repovore process --url https://github.com/owner/repo
uv run repovore process -i urls.txt               # from a file
5
Re-process existing repos 
uv run repovore process --all --force              # re-run full pipeline
uv run repovore process --all --stage summarize --force  # re-run one stage
uv run repovore process --all --from-stage score  # re-run from score onwards
The --all flag targets every repo already in the database — no need to supply URLs again.
6
Browse 
uv run repovore serve
Open http://localhost:8000 — the cards list shows a staleness banner when trending data hasn't been refreshed this week.
All CLI commands
